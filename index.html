Instalación

Sigue los siguientes comandos para instalar la aplicación:

  1. Para empezar, es necesario tener instalado el CLI de Angular de manera global. Además de NPM instalado en el computador que se desea realizar la instalación. Puedes instalar el CLI de Angular ejecutando npm install -g @angular/cli
  2. Copia o clona el repositorio de Gitlab.
  3. Una vez clonado el repositorio o extraído el archivo ZIP se verá la siguiente información.
    • Archivo angular.json - Contiene toda la información de Angular. Su versión, dependencias y más.
    • Carpeta coverage
    • Carpeta e2e
    • Archivo karma.config.js - Es un archivo de configuración. Más información en https://karma-runner.github.io/1.0/config/configuration-file.html
    • Carpeta node_modules - Contiene toda la información de los paquetes necesarios para que la aplicación de Gestión Curricular funcione correctamente.
    • Archivo package-lock.json
    • Archivo package.json
    • Archivo README.md - Contiene el instructivo de instalación en formato Markdown. Visible en Github o Gitlab.
    • Carpeta src - Contiene toda la información de la aplicación.
      • app - Vistas, componentes, módulos, Material Design, rutas y servicios.
      • assets - Imágenes de la aplicación
      • environments - Configuraciones de entorno en local y producción.
      • favicon.ico - Ícono de la aplicación en el navegador.
      • index.html – Archivo que verá el servidor al iniciarse.
      • main.ts
      • polyfills.ts
      • styles.css - Contiene los estilos generales de la aplicación.
      • test.ts
    • tsconfig.app.json
    • tsconfig.json
    • tsconfig.spec.json
    • tslint.json
  4. Después de ver todos estos archivos y carpetas (excepto el de node_modules) deberás abrir una terminal y ejecutar el comando npm install.
  5. Una vez instaladas todas las dependencias que necesita nuestro proyecto ejecutaremos ng serve o ng s (lo mismo pero simplificado) para iniciar nuestro servidor en local. Navega a http://localhost:4200/ para ver la aplicación en funcionamiento.

Estructura de las vistas

Las vistas de la aplicación para la Gestión Curricular utiliza el Framework Boostrap para su maquetado y para mejorar la adaptabilidad en todos los dispositivos.

Generalmente la estructura se vería así:

<!DOCTYPE html>
                        <html lang="en">
                        <head>

                        <!-- Your Title, Description, Stylesheets
                        ============================================= -->

                        </head>

                        <body data-spy="scroll" data-target=".idocs-navigation" data-offset="125">

                        <!-- Document Wrapper   
                        =============================== -->
                        <div id="main-wrapper"> 
                          
                          <!-- Header
                          ============================ -->
                          <header id="header" class="sticky-top">
                           ......
                          </header>
                          <!-- Header End --> 
                          
                          <!-- Content
                          ============================ -->
                          <div id="content" role="main"> 
                            
                            <!-- Sidebar Navigation
                            ============================ -->
                            <div class="idocs-navigation bg-light">
                             .....
                            </div>
                            
                            <!-- Content
                            ============================ -->
                            <div class="idocs-content">
                              <div class="container">
                                .......
                              </div>
                            </div>
                          </div>
                          <!-- Content end --> 
                          
                          <!-- Footer -->
                          <footer id="footer" class="section bg-dark footer-text-light">
                            <div class="container"> ...... </div>
                          </footer>
                          <!-- Footer end --> 
                          
                        </div>
                        <!-- Document Wrapper end --> 

                        <!-- JavaScript -->
                        </body>
                        </html>

If you need more information, please visit bootstrap site: https://getbootstrap.com

Personalización de cada vista

Todas las carpetas de vistas contienen la siguiente estructura:

  • Nombre de la vista dentro de la carpeta (Ejemplo stepper) src/app/fo
    • Archivo stepper.component.css - Contiene todos los estilos específicos para esa vista.
    • Archivo stepper.component.html - Contiene toda la estructura HTML que representa la vista.
    • Archivo stepper.component.ts - Contiene toda la lógica de la vista.

Entonces, para personalizar una vista podemos acceder al archivo stepper.component.css de cada vista o aplicar los cambios directamente en el archivo src/styles.css.

Code

Documentation and examples for displaying inline and multiline blocks of code

Inline code

Wrap inline snippets of code with <code>. Be sure to escape HTML angle brackets.

For example, <section> should be wrapped as inline.

For example, <code>&lt;section&gt;</code> should be wrapped as inline.

Code blocks

Use <pre>s for multiple lines of code. Once again, be sure to escape any angle brackets in the code for proper rendering. You may optionally add the .pre-scrollable class, which will set a max-height of 340px and provide a y-axis scrollbar.

For example:

<p>Sample text here...</p>
<p>And another line of sample text here...</p>

Have to use something like:

<pre><code>&lt;p&gt;Sample text here...&lt;/p&gt;
&lt;p&gt;And another line of sample text here...&lt;/p&gt;
</pre></code>

FAQ

Una FAQ es una lista de preguntas frecuentes (FAQ) y respuestas sobre un tema en particular.

¿Por qué no puedo editar las asignaturas si soy un docente?
Los docentes no podrán editar sus propias asignaturas. Las asignaturas son cargadas con su información básica y su Syllabus. Todas las sugerencias de modificación de Syllabus deberán pasar primero por un proceso de evaluación ante el Comité Curricular. Después de eso, solamente los docentes con el rol de "Gestor" o un rol superior podrán realizar las modificaciones.
¿Por qué los documentos que exporto dicen que no son válidos como Syllabus?
Solamente Admisiones y registro podrá exportar la información de Syllabus válido. Exportar un Syllabus y entregárselo a un estudiante no está permitido. Recuerda que todos los documentos tienen fecha de generación y usuario que lo generó.
¿Puedo descargar un Syllabus como PDF y enviárselo a un estudiante?
No, exportar documentos PDF con la información de un Syllabus no está permitido para los docentes. Este es un proceso que deberá realizar Adminisiones y registro después de que reciba el pago de derechos pecunarios por parte del estudiante.
¿Cada semestre los docentes son cambiados?
En ocasiones sí. Estas decisiones las tomará el Coordinador académico de cada programa.
